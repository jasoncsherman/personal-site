# Personal Site — Font Notes

A running record of typography decisions and the fonts Jason liked but didn't choose, kept for possible later use (other projects, future redesigns, accent uses).

_Last updated: 2026-06-15 (Phase 1 — style tile / type exploration)_

## The system (chosen 2026-06-15)

A **serif + mono** pairing, in the "Cool archive" direction:

- **Display / headings:** **Instrument Serif** (chosen). **Fraunces** is the agreed second-place / fallback.
- **Body / prose:** Inter (clean neutral sans, keeps reading calm).
- **Metadata / labels / nav:** **JetBrains Mono** — loved, core to the "reads tech" feel.

Rationale: serif for the artsy / written / editorial soul of the site, mono for technical precision. Editorial-meets-terminal. Max three type roles (per design brief).

### Instrument Serif — usage caveat (important)
Single weight (400 only) + high contrast. **Use it large** (page titles, section headers, hero, brand). For **small repeated titles** (book/writing list items ~16–18px) it can get delicate on dark — plan to use **Inter for those small UI-level titles**, keeping Instrument as the display face. Create hierarchy with size + color, NOT font-weight (no bold exists; never let the browser synthesize one). **Fraunces is the drop-in fallback** if Instrument ever feels too thin — it's variable-weight and works at all sizes.

## Chosen (final, 2026-06-15)

- **Headings (display, large):** Instrument Serif — page titles, section headers, hero, brand/name, pull quotes, and its italic. Weight 400 only; hierarchy via size + color, never faux-bold. Fallback: Fraunces.
- **Small list titles (book/writing/project rows ~16–18px):** Inter (medium/semibold) — decided after a scale test showed Instrument Serif gets too thin/recessive small. "Display serif for moments, sans for UI."
- **Body / prose:** Inter.
- **Metadata · labels · nav:** JetBrains Mono.

## Liked — kept in the running / for later use

- **Spectral** (serif) — refined modern serif; the current heading default and a strong candidate.
- **Space Grotesk** (sans) — clean, geometric, genuinely tech-y. Liked. A good sans option if we ever go sans headings.
- **Bricolage Grotesque** (sans) — grotesque with personality/character. Liked.
- **Geist** (sans) — Vercel's UI font. Jason had seen this before and liked it but couldn't recall the name — this was one of the "mystery fonts." Modern, precise, "dev tool" feel.

## Serifs evaluated (serif lab, 2026-06-15) — Jason liked ALL of these, saved for later

- **Instrument Serif** — high-contrast display serif; artsy, very modern editorial. **→ CHOSEN.**
- **Fraunces** — characterful old-style with modern optical sizing; warmer, more personality. **→ second place / fallback.**
- **Spectral** — refined modern serif; versatile workhorse across all sizes. Liked.
- **Source Serif 4** — clean contemporary serif; modern + highly readable. Liked.

## Rejected (and why)

- **JetBrains Mono as HEADINGS** — too mono; implies "I'm a developer," which is the wrong signal for a personal/editorial site. Great as metadata, not as display.
- **Inter Tight (sans headings)** — read as generic / "basic Arial." Not distinctive enough.
- **Warm editorial pairing** (Fraunces + Newsreader on cream) — the overall palette felt "library, old, dusty." Note: this was about the *brown/cream palette*, not Fraunces itself — Fraunces is being re-evaluated on the dark Cool-archive palette.
